## Branding Compliance Against CFO Standard
Required: logo, title, date range, generated-by, timestamp, page metadata.

- ✅ Accounting branded report renderer includes title/date/timestamp/logo path handling.
- ⚠️ Uniform “generated by” and page numbering is not guaranteed across all non-accounting report families.
- ⚠️ Export stack is fragmented outside accounting; some reports are print-only or export-only without parity.

## Parity Findings (UI vs Export)
- ✅ Accounting report CSV rows and print tables originate from the same report payload builders (lower mismatch risk).
- ⚠️ Procurement/inventory register exports are not universally routed through a single normalization layer, increasing parity risk.

## Remediation Recommendations
1. Create a shared `ReportExportService` for all modules (PDF/CSV/XLSX + branding contract).
2. Enforce one export metadata schema: `title`, `range`, `generated_by`, `generated_at`, `page`.
3. Add regression checks to compare UI totals vs exported totals for every report route.
